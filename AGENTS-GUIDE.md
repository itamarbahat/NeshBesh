## הסוכנים

### 👑 `@master` — מתזמר ראשי
**קובץ**: `.github/agents/master.agent.md`

**מתי להשתמש**: כשיש משימה מורכבת שנוגעת ביותר מתחום אחד.

**מה הוא עושה**:
- מנתח את הבקשה ומפרק אותה למשימות
- מפנה אוטומטית לסוכנים הנכונים (`@stylist`, `@logic`, `@fx`, `@debugger`)
- מסנכרן בין סוכנים — אם `@logic` משנה state, `@master` מוודא ש-`@stylist` יודע לרנדר את זה

**דוגמאות**:
```
@master אני רוצה להוסיף חוק 4:3 עם אנימציה מתאימה ו-UI
@master תתכנן לי את הפיצ'ר של bearing off
@master תבדוק שכל החוקים המיוחדים עובדים ביחד
```

**הגבלות**: לא עורך קוד בעצמו — רק מתכנן ומפנה.

### 🎨 `@stylist` — Agent A — עיצוב ו-UI
**קובץ**: `.github/agents/stylist.agent.md`

**מתי להשתמש**: שינויי עיצוב, לייאאוט, תמות, צבעים, responsive.

**תחום קבצים**: `neshbesh-app/src/components/`

**מה הוא עושה**:
- עיצוב הלוח, חלקים, וקונוסים
- הדגשות כחול (מהלכי ביניים) וירוק (יעדים סופיים)
- התאמה לטלפון מול טאבלט
- Overlays ופופאפים ללא חסימת משחק

**דוגמאות**:
```
@stylist תשנה את צבע הקונוסים לגוון עץ כהה יותר
@stylist תתאים את הלוח לאייפד ב-landscape
@stylist תוסיף UI ל-bearing off zone
@stylist יש overflow בצד שמאל של הלוח
```

**הגבלות**: לא נוגע במנוע המשחק (`engine/`, `store/`) או באנימציות.

### ⚙️ `@logic` — Agent B — מנוע משחק
**קובץ**: `.github/agents/logic.agent.md`

**מתי להשתמש**: כללי משחק, חוקי קוביות, state management, ניקוד.

**תחום קבצים**: `neshbesh-app/src/engine/`, `src/store/`, `src/types/`

**מה הוא עושה**:
- מערכת 2-קליק (בחירה → הדגשות → ביצוע → "נגעת נסעת")
- כל החוקים המיוחדים:
  - `1:2` — דילוג על תור
  - `4:5` — בחירת דאבל חופשית
  - `6:5` — Nesh Strike (בלוטים → בר + 2 מהלכים חופשיים)
  - `6:3` — שחק או הטל מחדש
  - `5:2` — תזוזה אחורה
  - `4:3` — הטלת קובייה → מספר צעדים אחורה
  - `5:1` — הטלת קובייה → קובע איזה דאבל לשחק
- דאבלים, 3 דאבלים רצופים = "הפיכת שולחן"
- ניקוד: רגיל(1), מארס(2), מארס טורקי(3), סטאר מארס(ניצחון מיידי)

**דוגמאות**:
```
@logic תממש את חוק 6:5 Nesh Strike
@logic יש באג במערכת bearing off — חלקים לא יורדים
@logic תוסיף ספירת דאבלים רצופים ל-store
@logic תממש ניקוד Turkish Mars
```

**הגבלות**: לא נוגע בקומפוננטות ויזואליות או באנימציות.

---

### 🎭 `@fx` — Agent C — אנימציות ואפקטים
**קובץ**: `.github/agents/fx.agent.md`

**מתי להשתמש**: אנימציות, אפקטי קול, הפטיקס, רצפים דרמטיים.

**תחום קבצים**: `neshbesh-app/src/animations/`, `src/audio/`

**מה הוא עושה**:
- **Reanimated**: אנימציות ביצועיות על UI thread
- **Moti**: אנימציות declarative פשוטות
- **expo-av**: אפקטי קול (אכילה, הטלת קוביות, ניצחון)
- **expo-haptics**: רטט פיזי
- אנימציות מיוחדות: Table Flip, Nesh Strike, dice glow

**דוגמאות**:
```
@fx תוסיף אנימציית הפיכת שולחן עם 3D rotation
@fx תוסיף צליל לאכילת חלק
@fx תוסיף glow מיוחד כשיוצא 6:5
@fx רטט הפטי כשזורקים קוביות
```

**הגבלות**: לא נוגע במנוע או ב-state. מתאם עם `@stylist` למיקום.

### 🔴 `@debugger` — Agent D — דיבאגר
**קובץ**: `.github/agents/debugger.agent.md`

**מתי להשתמש**: שגיאות, קריסות, בעיות ויזואליות, קונפליקטים.

**פרוטוקול עבודה**:
1. **סיווג**: Runtime / TypeScript / Visual / State desync
2. **מיפוי**: עקיבה לקובץ מדויק
3. **שורש**: זיהוי הבעיה המכנית
4. **תיקון**: Patch מינימלי
5. **אימות**: `npx tsc --noEmit`
6. **דוח**: מה נשבר, למה, ומה תוקן

**דוגמאות**:
```
@debugger: TypeError: board.map is not a function at Point.tsx:42
@debugger: Red screen — "Cannot read property 'length' of undefined"
@debugger: Moti ו-Reanimated קורסים ביחד אחרי עדכון
@debugger: הלוח נראה שבור באייפד — חלקים חורגים מהמסך
```

**הגבלות**: לא מבצע שינויים ספקולטיביים — תמיד עוקב עד שורש הבעיה.
